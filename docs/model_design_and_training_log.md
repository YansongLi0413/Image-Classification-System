## 1. 模型选型设计

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **架构多样性** | 覆盖 CNN 经典 → 残差 → 高效 → 密集 → Transformer 五大范式 |
| **公平可比** | 统一输入 224×224、统一优化器、统一数据划分 |
| **实用优先** | 优先使用 ImageNet 预训练权重（迁移学习），仅 Custom CNN 从头训练 |
| **可复现** | 固定随机种子(42)、记录所有超参数、保存训练历史 JSON |

### 1.2 五模型选择理由

| # | 模型 | 参数量 | 范式 | 选择理由 |
|---|------|:---:|------|----------|
| 1 | **Custom CNN** | ~45M | 经典卷积 | 基准模型，验证从头训练在中小数据上的表现 |
| 2 | **ResNet-50** | ~23.5M | 残差网络 | 工业标准基线，残差连接解决深层退化问题 |
| 3 | **EfficientNet-B1** | ~6.5M | 复合缩放 | 通过深度/宽度/分辨率复合缩放的 SOTA 高效架构 |
| 4 | **ViT-B/16** | ~86M | Transformer | Self-Attention 全局感受野，验证新范式在中小数据上可行性 |
| 5 | **DenseNet-121** | ~7.0M | 密集连接 | 特征复用最大化，抗过拟合优势，适合小样本场景 |

### 1.3 为什么用迁移学习

两个数据集各仅 ~7,000 张训练图片，102 个类别。从头训练大型网络（如 ViT 86M 参数）极易过拟合。

```
ImageNet (1.2M 图片, 1000 类) 预训练
        ↓
    骨干网络权重迁移（特征提取能力复用）
        ↓
   替换分类头 (num_classes=101/102)
        ↓
   两阶段微调适应目标数据分布
```

## 2. 训练框架设计

### 2.1 两阶段训练策略

```
【阶段一：分类头训练】(15 epochs)
  Backbone:  冻结 ❄️  — 保护预训练特征
  Classifier: 训练 🔥 — 学习目标类别映射
  LR:         0.001
  目的:       让分类头快速适应新任务

【阶段二：全网络微调】(35 epochs)
  Backbone:  解冻 🔥 — 微调特征提取层
  Classifier: 继续 🔥
  LR:         0.00005~0.0001  — 极低学习率防止破坏预训练特征
  目的:       适应目标数据分布细节
```

### 2.2 统一超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 输入尺寸 | 224×224 | 所有模型统一 |
| 批次大小 | 32 (ViT: 16) | ViT 因 86M 参数降批 |
| 优化器 | AdamW | 带解耦权重衰减的 Adam |
| 损失函数 | CrossEntropyLoss | 多分类标准 |
| 学习率调度 | CosineAnnealingLR | 平滑余弦退火 |
| 验证指标 | Accuracy (macro Precision/Recall/F1) | 宏平均，不受类别不均衡影响 |

### 2.3 数据增强（初版）

```python
# 训练时在线增强（不保存到磁盘）
RandomResizedCrop(224, scale=0.8-1.0)
RandomHorizontalFlip(p=0.5)
RandomRotation(±15°)
ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1)

# 验证/测试时
Resize(256) → CenterCrop(224)
```

## 4. 问题诊断与分析

### 4.1 Caltech-101 过拟合根因分析

```
现象:
  Train Acc → 99.9%  (几乎记住全部训练数据)
  Val Acc   → 90-92% (泛化能力不足)
  Gap       → 8-10%  (严重过拟合)

根因:
┌─────────────────────────────────────────────────┐
│ 1. 数据量不足                                     │
│    平均 ~60 张/类, 最少仅 31 张                    │
│    小类别训练集仅 ~21 张                           │
│                                                   │
│ 2. 类别极度不均衡                                  │
│    airplanes (800) vs inline_skate (31) = 25:1    │
│                                                   │
│ 3. 数据增强不足                                    │
│    初版增强对极小数据集保护不够                      │
│                                                   │
│ 4. 阶段二解冻过于激进                              │
│    全骨干解冻 + 仅有 1e-4 decay                    │
│    导致大模型快速过拟合小数据集                      │
│                                                   │
│ 5. 无类别权重                                     │
│    损失函数对各类别一视同仁                         │
│    大类别主导梯度，小类别被忽略                     │
└─────────────────────────────────────────────────┘
```

### 4.2 Oxford 102 为什么成功

```
Oxford 102 成功因素:
├── 数据更均衡: 36-234 张/类 (vs Caltech 31-800)
├── 图片质量高: 尺寸统一, 背景干净, 花卉居中
├── 预划分合理: 官方 train/valid 划分保证分布一致
└── 迁移学习有效: 预训练特征对花卉细粒度分类迁移良好
```

## 5. 优化改进方案

### 5.1 优化策略矩阵

| 维度 | 旧方案 | 新方案 | 预期效果 |
|------|--------|--------|----------|
| **数据增强** | RandomRotation(15°) + ColorJitter(0.2) | + **RandAugment**(ops=3, mag=12) + **RandomErasing**(p=0.3) | 大幅提升数据多样性 |
| **损失函数** | CrossEntropyLoss() | + **类别权重** + **标签平滑**(0.1) | 小类别被重视 + 防过置信 |
| **学习率调度** | ReduceLROnPlateau | **CosineAnnealingLR** | 更平滑收敛 |
| **阶段一权重衰减** | 1e-4 | **1e-3** (10×) | 更强正则化 |
| **阶段二学习率** | 1e-4 | **5e-5** (0.5×) | 保护预训练特征 |
| **阶段二权重衰减** | 1e-4 | **3e-3** (30×) | 强力防过拟合 |

### 5.2 代码改动

**文件**: `models/train.py`

```python
# Caltech-101 专属增强链:
if dataset_name == 'caltech101':
    aug_list.append(transforms.RandAugment(num_ops=3, magnitude=12))
    aug_list.append(transforms.RandomErasing(p=0.3, scale=(0.02, 0.15)))

# 类别权重 + 标签平滑:
if dataset_name == 'caltech101':
    class_weight = train_dataset.get_class_weights()
    label_smoothing = 0.1
    wd_phase1 = 1e-3    # ↑ 从 1e-4
    wd_phase2 = 3e-3    # ↑ 从 1e-4
    lr_phase2 = 5e-5    # ↓ 从 1e-4
```

**文件**: `models/trainer.py`

```python
# 损失函数 → 支持类别权重和标签平滑
criterion = nn.CrossEntropyLoss(
    weight=class_weight,
    label_smoothing=label_smoothing,
)

# 学习率调度 → CosineAnnealing
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=epochs, eta_min=lr * 0.01
)
```

### 6.1 优化过程中遇到的问题

| # | 问题 | 根因 | 修复 | 影响 |
|---|------|------|------|------|
| 4 | `AttributeError: 'Image' object has no attribute 'shape'` | RandAugment 在 ToTensor() 之前调用，输入是 PIL Image | 将 RandAugment 移到 ToTensor() 之后 | 全部模型 |
| 5 | `Only torch.uint8 image tensors are supported` | RandAugment 的 equalize/posterize/solarize 要求 uint8，但 ToTensor() 输出 float32 | 插入 float32→uint8→RandAugment→float32 转换 | 全部 Caltech-101 迁移模型 |

| 7 | `transforms.Lambda` 无法 pickle | Windows 多进程 DataLoader 无法序列化 lambda | 改用模块级函数 `_to_uint8` / `_to_float32` | 全部模型 |

## 9. 经验总结

### 9.1 关键教训

1. **小数据集 + 大模型 = 过拟合**
   - Caltech-101 仅 ~60 张/类 → 迁移学习也必须强正则化
   - 权重衰减 1e-4 → 1e-3 是缩小 Train-Val gap 的关键改进（gap 从 ~10% 降到 ~4%）

2. **数据质量 >> 模型复杂度**
   - Oxford 102 数据均衡(36-234/类) + 干净 → 全部模型轻松 98%+
   - Caltech-101 不平衡(31-800/类) → 即使强增强也只能到 93%

3. **RandAugment 对从零训练是毒药**
   - 对迁移模型有效提升 1-3%
   - 对 Custom CNN 直接摧毁训练（64% → 1.5%）—— 从零训练的模型需要「学会走再学跑」

4. **类别权重对从零训练也是毒药**
   - 27× 的权重范围导致梯度严重不稳定
   - 类别权重只适用于预训练模型微调阶段

5. **两阶段训练对小数据可能适得其反**
   - 所有模型最佳 epoch 都在阶段一早期（Epoch 2-20）
   - 阶段二解冻后 Val Acc 持续退化
   - 对 <100 张/类的数据集，只做阶段一可能更好

6. **torchvision 版本兼容性陷阱**
   - `RandAugment` 要求 uint8 输入（torchvision 0.21）
   - `IMAGENET1K_V2` 不是所有模型都有
   - ViT `SWAG_E2E_V1` 权重默认 384×384 输入
   - Lambda 函数在 Windows DataLoader 中无法 pickle → 用模块级函数

7. **CosineAnnealingLR vs ReduceLROnPlateau**
   - CosineAnnealing 适合迁移学习
   - ReduceLROnPlateau 对从零训练更友好（Custom CNN 从 64% 降到 57% 部分原因在此）

### 9.2 如果重来一次会怎么做

| 改进 | 说明 |
|------|------|
| **Caltech-101 不做阶段二** | 只跑阶段一(冻结)30-50 epochs，避免解冻导致的退化 |
| **更强的数据增强** | 加入 MixUp/CutMix，对小数据集效果更好 |
| **Custom CNN 单独配置** | 从零训练用 ReduceLROnPlateau + 轻增强 |
