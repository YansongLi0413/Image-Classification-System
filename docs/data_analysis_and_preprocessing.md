## 3. 数据预处理

### 3.1 Caltech-101 预处理

#### 3.1.1 类别处理

- **排除 BACKGROUND_Google**：这是一个背景噪声类，不是物体类别，与其他 101 类物体分类任务不一致
- **有效类别数**：101 类
- **标签映射**：创建 `class_name → class_id` (0-100) 的双向映射字典

#### 3.1.2 数据划分

采用**分层随机采样**（Stratified Shuffle Split），保证每个类别在 train/val/test 中的比例一致：

```
原始数据 (8,677 张, 101 类)
    │
    ├── 训练集 Train:  70% (6,073 张)  ─ 用于模型训练
    ├── 验证集 Val:    15% (1,302 张)  ─ 用于超参调优和早停
    └── 测试集 Test:   15% (1,302 张)  ─ 用于最终评估
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 随机种子 | 42 | 保证可复现 |
| 划分策略 | Stratified | 维持类别比例 |
| Train:Val:Test | 70:15:15 | 经典划分比例 |

#### 3.1.3 训练时数据增强（Data Augmentation）

针对 Caltech-101 类别不均衡和样本量小的问题，训练时使用**在线增强**（不保存到磁盘）：

| 增强操作 | 参数 | 目的 |
|----------|------|------|
| RandomResizedCrop | 224×224, scale=(0.8, 1.0) | 随机裁剪+缩放，增加空间不变性 |
| RandomHorizontalFlip | p=0.5 | 水平翻转，增加对称不变性 |
| RandomRotation | ±15° | 随机旋转，增加旋转不变性 |
| ColorJitter | brightness/contrast/saturation=0.2, hue=0.1 | 颜色扰动，增加光照不变性 |
| Normalize | mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225] | ImageNet 标准化（适配预训练模型） |

验证/测试时仅使用：
- `Resize(256)` → `CenterCrop(224)` → `ToTensor()` → `Normalize(ImageNet)`

## 5. 模型设计思路

### 5.1 设计原则

1. **多样性**：覆盖不同架构范式（CNN 经典 → CNN 残差 → CNN 高效 → CNN 密集 → Transformer）
2. **可比性**：统一输入尺寸 (224×224)、统一输出类别数 (101/102)、统一优化器与调度器
3. **实用性**：优先使用 ImageNet 预训练权重（迁移学习），同时保留一个从头训练的基准模型
4. **可复现性**：固定随机种子、记录所有超参数、保存训练历史

### 5.2 模型选择理由

| # | 模型 | 范式 | 选择理由 |
|---|------|------|----------|
| 1 | **Custom CNN** | 经典卷积（从头训练） | 作为基准模型，展示 CNN 基础设计能力；无预训练依赖，体现数据本身的学习难度 |
| 2 | **ResNet-50** | 残差网络（迁移学习） | 工业标准基线，残差连接解决了深层网络退化问题；50 层在精度与速度间取得平衡 |
| 3 | **EfficientNet-B1** | 高效缩放（迁移学习） | 通过复合缩放（深度/宽度/分辨率）达到 SOTA 效率；B1 版本适合 224×224 输入 |
| 4 | **Vision Transformer (ViT-B/16)** | Transformer（迁移学习） | 代表新范式：用 Self-Attention 替代卷积；验证 Transformer 在中小规模数据上的表现 |
| 5 | **DenseNet-121** | 密集连接（迁移学习） | 每层与前面所有层直连，特征复用最大化；抗过拟合能力强，适合小样本场景 |

### 5.3 为什么使用迁移学习？

两个数据集各自仅 ~7,000-8,000 张训练图片，102 个类别。从头训练大型网络（如 ResNet-50 的 23M 参数）极易过拟合。

**迁移学习策略**：
```
ImageNet (1.2M 图片, 1000 类) 预训练
        ↓
    骨干网络权重迁移
        ↓
   替换分类头 (num_classes=101/102)
        ↓
   第一阶段: 冻结骨干，仅训练分类头 (10-15 epochs)
        ↓
   第二阶段: 解冻骨干，低学习率微调 (15-30 epochs)
```

## 6. 模型架构详解

### 6.1 Custom CNN（基准模型）

```
输入: 224×224×3
    │
    ├─ Block 1: Conv(3→32)×2 → BN → ReLU → MaxPool(2)  → 112×112×32
    ├─ Block 2: Conv(32→64)×2 → BN → ReLU → MaxPool(2)  → 56×56×64
    ├─ Block 3: Conv(64→128)×2 → BN → ReLU → MaxPool(2) → 28×28×128
    ├─ Block 4: Conv(128→256)×2 → BN → ReLU → MaxPool(2) → 14×14×256
    ├─ Block 5: Conv(256→512)×2 → BN → ReLU → MaxPool(2) → 7×7×512
    │
    ├─ AdaptiveAvgPool2d(7×7)
    ├─ Flatten → FC(25088→1024) → ReLU → Dropout(0.5)
    ├─ FC(1024→512) → ReLU → Dropout(0.5)
    └─ FC(512→num_classes)
```

| 属性 | 值 |
|------|-----|
| 卷积层数 | 10 |
| 全连接层数 | 3 |
| 总参数 | ~45M |
| 设计理念 | 逐层增加通道数 (32→64→128→256→512)，空间尺寸逐步减半 |

### 6.2 ResNet-50（工业标准）

| 属性 | 值 |
|------|-----|
| 骨干网络 | ResNet-50 (ImageNet-1K V2 预训练) |
| 预训练参数 | 23.5M |
| 可训练参数 | 取决于是否冻结骨干 |
| 分类头 | Dropout(0.3) + FC(2048→num_classes) |
| 核心创新 | 残差连接 (Skip Connection)，解决梯度消失 |

### 6.3 EfficientNet-B1（高效缩放）

| 属性 | 值 |
|------|-----|
| 骨干网络 | EfficientNet-B1 (ImageNet-1K V2 预训练) |
| 预训练参数 | 6.5M |
| 分类头 | Dropout(0.3) + FC(1280→num_classes) |
| 核心创新 | 复合缩放 (深度×1.2, 宽度×1.1, 分辨率×1.15) |

### 6.4 Vision Transformer (ViT-B/16)

| 属性 | 值 |
|------|-----|
| 骨干网络 | ViT-B/16 (ImageNet-21K → ImageNet-1K 预训练) |
| Patch 大小 | 16×16 |
| 预训练参数 | 86M |
| 分类头 | Dropout(0.2) + FC(768→num_classes) |
| 核心创新 | Self-Attention 全局感受野，无卷积操作 |

### 6.5 DenseNet-121（密集连接）

| 属性 | 值 |
|------|-----|
| 骨干网络 | DenseNet-121 (ImageNet-1K V2 预训练) |
| 预训练参数 | 7.0M |
| 分类头 | Dropout(0.3) + FC(1024→num_classes) |
| 核心创新 | 密集连接 (每层接收前面所有层的特征图) |

### 6.6 模型规模对比

| 模型 | 总参数量 | 预训练 | 推理速度 | 抗过拟合 |
|------|:---:|:---:|:---:|:---:|
| Custom CNN | ~45M | ❌ | ⭐⭐⭐ | ⭐⭐ |
| ResNet-50 | ~23.5M | ✅ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| EfficientNet-B1 | ~6.5M | ✅ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| ViT-B/16 | ~86M | ✅ | ⭐⭐ | ⭐⭐ |

## 7. 训练策略

### 7.1 统一训练配置

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 输入尺寸 | 224×224 | 所有模型统一 |
| 批次大小 | 32 | 平衡内存与收敛速度 |
| 优化器 | AdamW | 带权重衰减的 Adam |
| 初始学习率 | 0.001 | 迁移学习模型；自定义 CNN 可用 0.01 |
| 权重衰减 | 1e-4 | L2 正则化 |
| 学习率调度 | ReduceLROnPlateau | 验证准确率停滞时减半 |
| 调度耐心值 | 5 epochs | 连续 5 epoch 无改善→降 LR |
| 早停耐心值 | 10 epochs | 连续 10 epoch 无改善→停止 |
| 最大 Epoch | 50 | 上限 |
| 损失函数 | CrossEntropyLoss | 多分类标准损失 |

### 7.2 分阶段训练

```
【阶段一：分类头训练】(Epoch 1-15)
  - 骨干网络: 冻结 ❄️
  - 分类头: 随机初始化，正常训练 🔥
  - 学习率: 0.001
  - 目的: 让分类头适应新任务

【阶段二：全网络微调】(Epoch 16-50)
  - 骨干网络: 解冻 🔥
  - 分类头: 继续训练 🔥
  - 学习率: 0.0001 (降低10倍)
  - 目的: 微调整个网络以适应目标数据分布
```

> 📌 Custom CNN 无需分阶段，从头直接训练50轮。

### 7.3 类别不平衡处理

针对 **Caltech-101 极不均衡**问题：

| 策略 | 实现 |
|------|------|
| 分层采样 | `train_test_split(stratify=labels)` 确保每类在三个划分中比例一致 |
| 数据增强 | 训练时在线增强，特别是对小类别更多增强 |
| 类别权重 | 可用 `Dataset.get_class_weights()` 计算权重，传入损失函数 |

## 8. 评估指标体系

### 8.1 核心指标

| 指标 | 公式 | 意义 |
|------|------|------|
| **Accuracy** | (TP+TN) / Total | 整体正确率（主要指标） |
| **Precision** | TP / (TP+FP) | 预测为正例中真正的正例比例（macro avg） |
| **Recall** | TP / (TP+FN) | 真正例中被正确识别的比例（macro avg） |
| **F1 Score** | 2×P×R / (P+R) | 精确率和召回率的调和平均（macro avg） |

> 使用 `macro average` 计算多类别 Precision/Recall/F1，即先计算每类指标再取平均，不受类别不均衡影响。

### 8.2 辅助分析

| 分析维度 | 方法 |
|----------|------|
| 混淆矩阵 | 可视化每类预测分布，发现易混淆类别对 |
| Top-5 Accuracy | 真实标签在 top-5 预测中的比例 |
| 训练曲线 | Loss 和 Accuracy 的 epoch 变化曲线 |
| 推理时间 | 单张图片的平均推理耗时 (ms) |
| 模型大小 | 存储模型权重的磁盘占用 (MB) |

### 8.3 目标准确率

| 数据集 | 保底目标 | 期望目标 |
|--------|:---:|:---:|
| Caltech-101 | 90% | 95% |
| Oxford 102 | 90% | 95% |

## 9. 模型对比分析方案
